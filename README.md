# Book Finder Assistant (Telegram)

Telegram-бот для поиска книг по **тексту** и **фото обложки**.

Главная идея: пользователь отправляет описание/цитату или картинку, бот определяет произведение и автора, затем ищет в **Флибусте** (приоритет), а если не уверен — делает fallback в **Google Books**.

## Какие задачи выполняет проект

### 1) Поиск книги по тексту
- Принимает обычный текст (без обязательного `/find`) или `/find <запрос>`.
- Использует Gemini для извлечения структурированного запроса:
  - `query`
  - `title` (если уверенно)
  - `author` (если уверенно)
  - `confidence`
- Строит несколько попыток поиска и проверяет совпадения по скорингу.
- Приоритетно ищет во Флибусте; если не сработало — обращается к Google Books.

### 2) Поиск книги по фото
- Принимает фото в Telegram.
- Скачивает файл из Telegram и отправляет в Gemini Vision.
- Извлекает вероятные `title/author/isbn/evidence`.
- Проверяет совпадение во Флибусте и формирует ответ со ссылками скачивания.
- Если уверенного совпадения нет — fallback в Google Books.

### 3) Интеграция с Флибустой
- Поиск книг и авторов через `flibusta-api`.
- Получение метаданных книги (жанры/описание).
- Генерация ссылок на форматы (MOBI/EPUB).
- Нормализация ошибок сети/таймаутов/403/429 в user-friendly сообщения.

### 4) Интеграция с Google Books (fallback)
- Поиск по сформированному запросу.
- Выдача лучшего совпадения и ссылки для открытия книги.
- Используется, когда Флибуста не дала уверенный результат.

### 5) Скоринг и матчинг результатов
- Нормализация текста (регистр, символы, пунктуация).
- Сравнение по exact/partial совпадению title и author.
- Дополнительный token-overlap скоринг для неполных запросов.
- Порог принятия для решения «совпадение/не совпадение».

### 6) Кэширование
- Кэш ответов по SHA-256 хэшу изображения.
- Уменьшает повторные вычисления и ускоряет ответ на одинаковые фото.

### 7) Режимы отладки
- `/raw on|off` — показывает сырой JSON Vision-ответа.
- `/fdebug on|off` — дебаг запросов к Флибусте.
- `/gdebug on|off` — дебаг Gemini для текстового извлечения.

### 8) UX в Telegram
- Reply-keyboard меню:
  - `🔎 По тексту`
  - `📷 По фото`
  - `ℹ️ Помощь`
- Регистрирует команды бота через `setMyCommands`.

### 9) Контроль доступа пользователей (onboarding)
Проект поддерживает модель доступа с owner-контролем:
- Новые пользователи отправляют запрос доступа кнопкой.
- Владелец получает заявку с кнопками Approve/Reject.
- Есть ручные команды владельца:
  - `/users`
  - `/allow <user_id>`
  - `/deny <user_id>`
- Состояние доступа сохраняется в `data/access.json`.

### 10) Ограничение по топику/треду
- Можно ограничить обработку сообщений конкретным `ALLOWED_THREAD_ID`.

## Быстрый запуск
1. Установить зависимости:
```bash
npm install
```

2. Создать `.env`:
```env
BOT_TOKEN=...
GEMINI_API_KEY=...
GOOGLE_BOOKS_API_KEY=...
OWNER_ID=...
# optional:
# DATA_DIR=./data
# FLIBUSTA_BASE_URL=https://flibusta.is
# ACCESS_FILE=./data/access.json
# KINDLE_FILE=./data/kindle.json
# RESEND_API_KEY=...
# RESEND_FROM=onboarding@resend.dev
```

3. Запустить:
```bash
npm start
```

## Persistent storage (Railway Volume)
Если деплои сбрасывают данные (доступы/Kindle email), подключи Volume и задай:
```
DATA_DIR=/data
```
Тогда файлы `access.json` и `kindle.json` будут сохраняться между деплоями.

## Основные команды
- `/start` — старт и меню
- `/help` — помощь
- `/menu` — показать клавиатуру
- `/find <текст>` — явный текстовый поиск
- `/raw on|off`
- `/fdebug on|off`
- `/gdebug on|off`
- `/users` (owner)
- `/allow <user_id>` (owner)
- `/deny <user_id>` (owner)
